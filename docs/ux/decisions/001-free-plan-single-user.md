# UXDR 001: Freeプランのユーザー数を1人にする

Status: Adopted  
Date: 2026-06

## 状況

Costoraはマルチテナント（組織/個人）対応が強みだが、Freeプランで組織試用を許可すると
「CFOにも見せたい」というアップグレード動機が機能しなくなる。
また、複数ユーザーのサポートコストは規模に依存するため、Freeでの解放はリスクが高い。

## 決定

Freeは1人（個人利用向け）とし、組織での試用はStarterの14日間無料トライアルへ誘導する。

LP・Settings画面のアップグレード訴求では「チームで使いたい場合はStarter」と明示する。

## 根拠

- 「CFOにも見せたい」が[アップグレード動機シナリオ④](../../business/revenue-model.md#freeプランの設計思想)に該当し、1ユーザー制限が壁として機能する
- 競合（Finout・CloudZero・Apptio）もFreeでの複数ユーザーは提供していない
- Starter 14日トライアルで組織試用の障壁を下げることで、Freeとのギャップを補える

## トレードオフ

- 組織での初期試用が難しくなる（Starterトライアルに誘導が必要）
- LP上で「個人利用向け」と明示しないと誤解を招く
- 将来的に「チームFree」プランを作る場合、このUXDRの改訂が必要
