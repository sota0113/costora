# Costora UX原則

設計・実装の意思決定時にここに立ち戻る。新しいUIコンポーネントや操作フローを作るとき、これら原則に反していないか確認する。

---

## 1. 5分でセットアップ完了

サインアップから最初のサービスを接続してコストが表示されるまで、5分以内に完了できること。
入力項目・説明文・エラーメッセージは「最初のコストが見えるまで」を最短にする視点で設計する。

> 測定基準: `docs/ux/flows/onboarding.md` のステップ数と所要時間

## 2. 非エンジニア経営者が読める

CFO・経営企画担当が日常的に使う前提で語彙・構造を設計する。
「タグ」「クレデンシャル」「API キー」のような技術用語は、入力必須箇所でも補足説明を添える。
KPIは金額・部門・前月比で表示し、raw データはドリルダウンの奥に置く。

## 3. やめられない体験を先に作る

価値体験の前に制限を当てない。
部門管理・6方式の按分はFreeプランでも全開放する（使えることで価値を理解させてから壁を置く）。
制限に達したとき、壁の直前に「これができる」を見せてからアップグレードを促す。

## 4. 認証情報を見せない

APIキー・シークレット等はUIに平文で表示しない。入力後は必ずマスク表示（`••••••••`）にとどめる。
Settings画面での表示・コピー機能も同様。復号値がブラウザに露出するタイミングをゼロにする。
この原則はセキュリティ要件であると同時に、「あなたの認証情報を大切に扱う」という信頼シグナルでもある。
